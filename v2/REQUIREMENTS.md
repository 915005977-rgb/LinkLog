# LinkLog V2 需求摘要

> 由画统筹基于用户需求整理 · 2026-06-13

---

## 1. Surface（场景）

### 产出物清单

| # | 产出物 | 形态 | 设备优先级 | 文件名 |
|---|--------|------|-----------|--------|
| 1 | 升级版联系人详情页 | 移动端 HTML 原型 | Mobile First | `detail-v2.html` |
| 2 | 人物资料表格 | PC 端 HTML 原型 | Desktop Only | `people-table.html` |
| 3 | 场域洞察 Dashboard | PC 端 HTML 原型 | Desktop Only | `field-insights-v2.html` |

### 产出物说明

**1. 升级版联系人详情页（detail-v2.html）**
- 替换现有 MVP 的详情页
- 保留现有：人物身份卡（72px头像+标签）、核心四问（Q1Q2展开/Q3Q4折叠）、AI破冰卡片
- 新增：深度档案区块，包含12个维度字段
- 交互：渐进式披露，默认展示核心信息，展开查看完整档案

**2. 人物资料表格（people-table.html）**
- PC 端宽表格视图，展示所有联系人的多维度信息
- 参考：people-profiles-table.html 的结构
- 核心功能：sticky 首列（人物名+角色+标签）、多列维度横向对比、搜索、标签筛选、可展开完整档案
- 数据：8-10 位脱敏联系人

**3. 场域洞察 Dashboard（field-insights-v2.html）**
- 统计概览 + 标签分布 + 关系洞察
- 参考：field-insights.html 的结构
- 核心功能：Hero 区（标题+4格统计）、筛选栏（搜索+标签chip）、标签分布可视化、关系洞察卡片
- 数据：基于脱敏联系人的聚合统计

## 2. Audience（受众）

### 主要用户
- **个人关系管理者**：参加行业活动后需要快速整理人脉的职场人
- **一人公司/超级个体**：需要系统性管理人脉资源和商业机会的创业者
- **社交活跃者**：认识人多但缺乏结构化记录和连接策略的人

### 使用场景
- 活动后快速录入和整理联系人信息
- 定期回顾人脉网络，发现连接机会
- 从群体数据中洞察行业趋势和资源缺口

---

## 3. Tone（调性）

### 视觉方向
**Modern Minimal × Soft Warm — 继承 LinkLog 现有设计系统**

- 温暖包裹感：暖灰底色 + 琥珀橘强调 + 冷蓝紫 AI 智能感
- 呼吸感优先：渐进式披露，不一次展示所有信息
- 温度梯度叙事：人（暖灰）→ 情感连接（琥珀橘）→ AI 辅助（冷蓝紫）

### 关键调性词
- 专业但不冰冷
- 有温度的效率工具
- 数据丰富但不压迫
- 个人化、有人情味的 CRM

---

## 4. Brand Context（品牌上下文）

### 品牌资产
- **品牌名**：LinkLog — 连接日志 · 关系复利
- **设计系统**：已建立完整 DESIGN.md（Notion 定制系统）
- **色彩温度梯度**：暖灰 #F8F6F3 → 琥珀橘 #E8915A → 冷蓝紫 #7C6AE8
- **CSS 变量体系**：完整的色彩/间距/圆角/阴影/动效 token
- **现有 MVP**：移动端 6 页 + PC 端体验页

### 产品进化路径
**记录个体 → 积累群体 → 洞察场域**
- Phase 1（已完成）：单个联系人的录入和管理
- Phase 2（本次）：深度档案 + 群体数据表格视图
- Phase 3（本次）：场域洞察和关系网络

---

## 5. Scale（规模）

| 维度 | 详情 |
|------|------|
| 页面数 | 3 个 HTML 文件 |
| 联系人数据 | 8-10 位脱敏样本 |
| 字段维度 | 12+ 维度/人（背景/产品/卡点/话题等） |
| 代码规模 | 每文件预计 2000-3000 行 HTML/CSS/JS |
| 外部依赖 | 零依赖，纯 HTML/CSS/JS |

## 附录：深度档案字段定义

### 基础信息（始终可见）
| 字段 | 说明 | 来源 |
|------|------|------|
| 姓名+公司/角色 | 身份标识 | 现有 |
| 标签/备注 | AI 生成标签 | 现有（增强） |

### 身世线（默认展开）
| 字段 | 说明 | 类型 |
|------|------|------|
| 背景/履历 | 职业和教育背景 | 用户输入 |
| 过去做过什么 | 历史经历 | 用户输入 |
| 现在在做什么 | 当前主业/项目 | 用户输入 |

### 价值线（默认展开）
| 字段 | 说明 | 类型 |
|------|------|------|
| 产品/服务 | 在卖什么/提供什么 | 用户输入 |
| 商业化进度 | 验证到哪个阶段 | 用户输入 |
| 优势/可分享 | 核心能力和可分享内容 | 用户输入 |

### 连接线（默认折叠）
| 字段 | 说明 | 类型 |
|------|------|------|
| 当前卡点 | 最大瓶颈 | 用户输入 |
| 想链接资源 | 需要什么资源/人 | 用户输入 |
| 与你的关联 | 对你的价值 | 用户输入 |
| 你能提供什么 | 你能帮到什么 | AI 辅助生成 |
| 你能学什么 | 可学习的点 | AI 辅助生成 |

### 行动线（AI 增强区）
| 字段 | 说明 | 类型 |
|------|------|------|
| 现场话题 | AI 生成的破冰话术 | AI 生成（冷蓝紫标识） |
| 会后动作 | AI 建议的跟进步骤 | AI 生成（冷蓝紫标识） |

## 附录：脱敏数据规范

- 真实人名 → 化名（如 Karen → 林社长，Cherry → 陈老师）
- 真实公司名 → 行业+规模描述（如 微软 → 顶级外企，SoloNest → 创业社区平台）
- 真实产品名 → 泛化描述（如 clickmab.com → AI 设计平台）
- 保持数据的结构完整性和逻辑自洽
- 保持角色多样性和标签丰富度
