# LinkLog — 连接日志 · 关系复利

> 把每次连接变成可复利的关系资产

## 项目介绍

LinkLog 是一款**个人 CRM 产品**，核心理念是「连接日志 · 关系复利」——帮助用户把每次社交连接转化为可追溯、可复利的关系资产。

与传统 CRM 不同，LinkLog 不只记录名片信息，而是引导用户深度思考**为什么连接、想学什么、能提供什么、怎么破冰**，通过「核心四问」框架让关系从「认识」升级为「深度连接」。

### 核心功能

- **AI 标签体系**：基于自我介绍，AI 自动推荐行业方向、关系类型、行为标签
- **联系人管理**：记录姓名、角色、公司、星座、MBTI 等信息
- **核心四问**：引导用户深入思考连接意义（为什么连接 / 想学什么 / 能提供什么 / 破冰开场白）
- **AI 破冰建议**：基于星座 + MBTI 性格特质，AI 生成个性化破冰策略
- **自定义标签**：支持用户自定义标签，灵活适配不同社交场景

## 设计理念

### 视觉方向：Modern Minimal × Soft Warm

**简约骨架 + 温暖血肉** — 结构严谨如精密仪器，但每一个触感都带着温度。

### 温度梯度叙事

设计系统通过色彩温度构建认知层次：

| 温度 | 色系 | 含义 | 代表色 |
|------|------|------|--------|
| 🟠 暖 | 琥珀橘 | 人的情感、关系、温度 | `#E8915A` |
| ⚪ 中性 | 暖灰 | 内容、留白、呼吸感 | `#F8F6F3` |
| 🔵 冷 | 冷蓝紫 | AI 智能、机器生成 | `#7C6AE8` |

**人 → 情感 → 机器**：从暖到冷的温度叙事，帮助用户直觉区分内容来源。

### 视觉三原则

1. **呼吸感优先**：任何页面至少 40% 留白
2. **温度梯度**：暖灰底色 → 琥珀橘强调 → 冷蓝紫 AI
3. **灵动手感**：每个状态变化都有物理隐喻（弹跳、滑出、渐入）

## 页面流程

原型包含 **首页、联系人详情页、添加联系人流程（4步）** 共 6 个页面，完整覆盖核心使用场景：

```
首页(Dashboard)
    │
    ├─ 点击卡片 → 联系人详情页
    │
    └─ 点击 FAB / Tab「添加」 → 添加联系人流程
         │
         ▼
    Step 1: 标签体系设置
         │
         ▼
    Step 2a: 基本信息 + 标签选择
         │
         ▼
    Step 2b: 核心四问 + AI 破冰
         │
         ▼
    Step 3: 保存成功确认 → 可跳转详情页或返回首页
```

### 首页（Dashboard）

首页是用户的主操作界面，包含以下功能区域：

- **问候区**：根据当前时间自动切换问候语（早上好/中午好/下午好/晚上好），显示用户头像
- **统计感知条**：展示已记录人数、本周新增、待跟进等关键指标
- **搜索栏**：支持按姓名、公司、标签搜索联系人，滚动时自动吸顶并添加阴影
- **筛选按钮**：打开筛选 Bottom Sheet，按关系维度/场景维度筛选
- **联系人列表**：卡片式列表，展示头像（关系维度色）、姓名、角色·公司、标签预览
- **FAB 按钮**：右下角悬浮添加按钮，点击进入添加联系人流程
- **底部 Tab Bar**：首页 / 添加 / 设置 三个入口
- **空状态**：无联系人时显示引导页面

### 联系人详情页

从首页点击联系人卡片进入，展示完整的联系人信息：

- **毛玻璃导航栏**：返回按钮 + 更多操作（编辑/分享/删除）
- **人物身份区**：大头像（关系维度色）、姓名、角色·公司、星座/MBTI/公司标签、全部标签（超出6个可折叠）
- **核心四问回顾区**：显示已记录的四个问题及回答，默认展开前两问，后两问可展开/收起
- **AI 破冰建议区**：虚线边框 + 冷蓝紫底色，包含开场建议（引用式）和话题方向列表，支持重新生成和复制
- **删除确认**：二次确认弹窗，防止误删

### Step 1：标签体系设置
- 用户填写「自我介绍」
- 点击「AI 推荐标签体系」按钮，AI 根据自我介绍生成专属标签
- 标签分三组：行业方向、关系类型、行为标签
- 支持自定义标签（输入框 + 添加按钮）
- 标签以逐个渐入动画出现，间隔 80ms

### Step 2a：基本信息 + 标签选择
- 填写联系人基本信息：姓名（必填）、角色、公司
- 选择星座和 MBTI（为后续 AI 破冰做准备）
- 选择行业方向标签和关系类型标签（点选高亮）

### Step 2b：核心四问 + AI 破冰
- 四个问题以卡片滑动形式逐个呈现：
  1. 为什么连接？
  2. 想从TA身上学什么？
  3. 你能为TA提供什么？
  4. 破冰开场白
- 每回答一题，卡片上滑飞出，下一张卡片滑入
- 四问全部完成后，AI 破冰提示卡片展开（结构级 reveal 动画）
- AI 基于星座 + MBTI 生成个性化破冰建议

### Step 3：保存成功确认
- 成功图标弹出动画
- 显示联系人姓名
- 三个操作：继续添加 / 查看联系人 / 返回首页

## 交互亮点

### 灵动动画
- **标签渐入**：AI 生成标签逐个渐入（间隔 80ms），从下方 8px 滑入 + 透明度渐变
- **卡片飞出**：核心四问回答后卡片上滑飞出（`translateY(-120%) scale(0.9)`）
- **按钮弹跳**：CTA 按钮点击时弹性缩放（`scale(0.97)`，`ease-spring` 曲线）
- **输入聚焦**：输入框获焦时微放大（`scale(1.005)`）+ 焦点环
- **搜索吸顶**：首页搜索栏滚动时自动吸顶 + 阴影提升
- **问候区淡出**：首页向下滚动超过 40px 时，问候区渐隐上移（`opacity: 0; translateY(-10px)`）
- **卡片点击反馈**：联系人卡片按下时微缩（`scale(0.98)`），背景色变化
- **页面转场动画**：首页→详情页左右滑动切换（`translateX`，300ms）

### AI 交互模式
- **结构级隐藏**：AI 破冰卡片初始 `max-height: 0; opacity: 0; overflow: hidden`，完成后 reveal
- **温度区分**：AI 元素使用冷蓝紫（`#7C6AE8`）+ 虚线边框 + 浅底，与暖色用户内容形成对比
- **实时更新**：星座 / MBTI 变更后，AI 破冰建议实时重新生成

### 体验细节
- **触摸目标**：所有标签最小高度 32px，标签间距 12px，符合移动端触摸标准
- **焦点可见**：所有按钮支持 `focus-visible` 焦点环，提升键盘可访问性
- **减弱动效**：尊重 `prefers-reduced-motion` 系统设置
- **Toast 提示**：替代 `alert()`，顶部圆角胶囊式提示条

### 步骤指示器
- 三点式步骤指示器（活跃态横向拉伸为 24px 胶囊）
- 已完成步骤变绿（`#4CAF7D`），连接线同步变色
- 步骤切换时横向滑动过渡（`translateX`，400ms）

## 技术说明

- **零依赖**：不引用任何 CDN、外部字体、外部样式表
- **单文件**：所有 CSS（`<style>`）、JS（`<script>`）、图标（inline SVG）内联于 `index.html`
- **系统字体栈**：Inter → SF Pro Display → Noto Sans SC → PingFang SC → Microsoft YaHei，保证各平台中文渲染
- **CSS 变量**：所有设计令牌通过 CSS 变量定义，一处修改全局生效
- **设计基准**：iPhone 15（393 × 852px）
- **关系维度色**：通过 CSS `data-relation` 属性驱动头像配色，不硬编码在 JS 中
